# 标注规范

本项目假设已经有约 200 张 LabelMe 标注图，每张图包含：

- 1 个声门区域旋转框：标签名默认 `声门区域`
- 3 个点：标签名默认 `前联合`, `左后方中点`, `右后方中点`

默认点位顺序：

| 点 | 含义 |
| --- | --- |
| `前联合` | 声门前联合点 |
| `左后方中点` | 左侧后方中点 |
| `右后方中点` | 右侧后方中点 |

如果实际标注采用不同名字或顺序，只改 `configs/keypoints.yaml`，不要混用多套规则。

## 声门区域框

LabelMe 中的 `声门区域` 可以是 `oriented_rectangle`。转换到 YOLO-Pose 时，训练标签仍使用它的 axis-aligned 外接框，因为 Ultralytics pose 标签只支持普通 bbox；原始旋转框四角会额外写入 `data/yolo_pose/roi_polygons/{split}.jsonl`，用于评价“覆盖人工 ROI 至少 87%，面积尽量小”。推理阶段的最终可用框不是这个 axis-aligned 外接框，而是由 3 个关键点几何生成的 `final_box_polygon`：一条边平行于前联合夹角的角平分线，左右宽度分别由两侧后方点到角平分线的投影决定。

## 训练前自动修正

转换脚本会检查训练 bbox 是否包含 3 个关键点。若关键点在框外，会用：

```text
union(manual_bbox, keypoint_bbox_with_margin)
```

生成训练框，避免 YOLO-Pose 同时收到“框在这里”和“点在框外”的矛盾信号。

## 翻转增强

主分支默认关闭水平/垂直翻转，因为翻转会改变左右或前后点位含义。只有在明确写好 `flip_idx` 和点位重排规则后，才建议打开。

这批标注对旋转不敏感，但对左右语义敏感：前联合在上方时，图像右侧的后方点可能是解剖学左后方中点。因此训练默认不做翻转，几何后处理本身只依赖两侧点形成的夹角，对左右点互换保持稳定。

## 黑边增强

当前黑边增强数据集不是整张图都有有效信息。黑色 padding 是为了让最终旋转矩形 ROI 不被原图边界截断；训练和评价应使用增强后的图像尺寸与坐标，不要把黑边裁回去。
